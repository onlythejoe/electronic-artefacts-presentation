Infrastructure

Surface projects. Context defines semantics.

The system is organized as a canonical core graph. Surface turns the system into a public projection, context stores the registry, intents route actions, and modules enrich the middle layer.

Surface → Context → Intents → Modules → Core

Core is the first canonical node. Inside it, only nodes and edges define topology.

Surface Public projection Pages, UI, visible output.
Context Global registry Semantics, policy, state.
Intents Action corridor User & System actions routed into the graph.
Modules Middle layer enrichment Capabilities attached to canonical nodes.
Core Nodes and edges only Topology remains canonical.

Build

Surface Studio designs surfaces. Programs wire nodes. Context gives them type.

SPACE is built from surfaces. Surface Studio composes the public layer, programs orchestrate node relations, context defines what a node is in context, modules enrich the middle layer, and Assets lets any file type land on a node.

Domotics can drive several things at once. The core remains canonical; everything else attaches around it.
